# PRD & Implementation Plan: 인천소방학교 이전 신축공사 내역서 대시보드

## 1. 프로젝트 개요 (Product Requirements Document)

### 1.1 배경 및 목적
인천시는 도심 내 위치한 인천소방학교를 강화도로 이전하여 최적의 훈련시설(119 SQUARE)을 건립할 계획입니다(총 사업비 약 430억). 기존의 방대하고 파편화된 엑셀 내역서(건축, 기계, 조경, 토목) 데이터를 데이터베이스화하고, 이를 누구나 쉽게 검색하고 분석할 수 있는 **웹 기반의 시각적 대시보드**로 구축하는 것이 본 프로젝트의 핵심 목표입니다.

### 1.2 핵심 사용자 요구사항
- 분산된 엑셀 데이터를 하나의 통합 DB로 검색할 수 있어야 함.
- 텍스트 가독성이 높고 쾌적한 스크롤이 보장된 상세 데이터 표 제공.
- 건설공사 표준분류체계(20개 컬럼)를 준수하며, 특히 기계내역서 내부의 기계, 전기, 소방 내역을 지능적으로 분리할 것.
- 시각적 디자인: **Tilt-shift / Miniature / Toy Package** 스타일을 적용하여, 정보의 무게감을 덜고 세련되고 직관적인 "장난감 프레젠테이션 슬라이드" 느낌을 줄 것.

---

## 2. 시스템 아키텍처 및 기술 스택

- **데이터베이스 (Database):** `SQLite3` (단일 파일 기반, 빠르고 가벼운 조회 환경 구성)
- **데이터 파이프라인 (ETL):** `Python (Pandas)` - 엑셀 파싱 및 자동 키워드 분류 알고리즘 적용
- **백엔드 (Backend):** `FastAPI` - 초고속 비동기 REST API (검색, 페이징, 데이터 집계)
- **프론트엔드 (Frontend):** `React` + `Vite`
  - **시각화:** `Recharts` (Bar Chart, Pie Chart)
  - **스타일링:** `Vanilla CSS` + Glassmorphism + Backdrop-filter Blur
  - **아이콘:** `Lucide-react`

## 3. 구현된 핵심 기능 및 파이프라인 (Implementation History)

### Phase 1: 데이터 통합 및 표준분류체계 마이그레이션 (ETL)
* **20개 표준 컬럼 도입:** 기존 단순 스키마를 폐기하고, `WHERE1_프로젝트`, `HOW1_공사`, `HOW2_대공종`, `R7_재료비_금액` 등 20개의 전문적인 표준 건설공사 분류체계로 데이터베이스(`project_costs.db`)를 재구축했습니다.
* **기계내역서 스마트 분할:** 단일 파일로 제공된 기계 엑셀 문서 내에서 '전선', '케이블', '소화', '감지기' 등의 품명 키워드를 분석해 **기계설비 공사, 전기 공사, 소방 공사** 데이터를 자동으로 분류하고 적재했습니다. (총 4,483건 추출 완료)

### Phase 2: 고성능 백엔드 API (Backend)
* `GET /api/summary`: 선택된 공종과 검색어에 맞춰 실시간으로 동적 합계(총 사업비 및 공종별 예산)를 계산해 반환합니다.
* `GET /api/items`: 수천 건의 상세 내역을 빠르게 렌더링하기 위해 SQL 레벨에서의 페이징(Pagination) 및 부분 일치 검색(LIKE Query)을 완벽 지원합니다.

### Phase 3: 장난감 미니어처 테마 UI/UX (Frontend)
* **Tilt-shift & Blur 효과:** AI로 생성된 미니어처 느낌의 장난감 도시 사진을 배경에 깔고, CSS `mask-image`와 `backdrop-filter`를 활용해 화면 위아래에 강한 심도(Blur) 효과를 주어 틸트 시프트 렌즈 느낌을 재현했습니다.
* **프로젝트 개요 슬라이드:** 대시보드 최상단에 대지위치, 연면적, 층수 등 요약 텍스트를 담은 글래스모피즘(Glassmorphism) 패널을 배치했습니다.
* **비주얼 데이터 차트:** 채도 높은 원색(Red, Blue, Yellow 등)을 활용해 막대그래프(비용 분포)와 파이 차트(예산 점유율)를 나란히 배치하여 장난감 패키지 같은 귀여움과 명료함을 잡았습니다.
* **와이드 데이터 테이블:** `수량`, `단가`, `재료비`, `노무비`, `경비`, `합계금액` 등 수많은 세부 정보가 한눈에 들어오도록 가로 스크롤을 도입하고 최소 높이(`min-height`)를 보장하여 스크롤 짤림 버그를 완전히 해결했습니다.

## 4. 향후 유지보수 및 개선 방향 (Next Steps)
1. **운영 배포 환경 구성:** Vercel(프론트엔드) 및 AWS/Render(백엔드 API) 연동 배포.
2. **엑셀 내보내기 기능 추가:** 필터링된 대시보드의 표 데이터를 다시 Excel 파일로 다운로드할 수 있는 기능 추가 고려.
3. **데이터 업데이트 자동화:** 향후 새로운 내역서 파일이 업로드될 때마다 `.raw_data` 폴더를 감지하여 자동으로 `etl.py` 파이프라인을 돌리는 스크립트 작성.
